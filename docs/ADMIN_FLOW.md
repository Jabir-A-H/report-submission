# Admin Flow

## Overview
This document outlines the workflows specific to Administrators. Admins have overarching control of the platform, user management, and city-wide data aggregation.

## 1. User Management & Approvals
Because the system relies on strict permissions, new registrations are gated.
- **Route**: `/admin/users`
- **Action**: Admins view a list of users with `active = false`. They can review the user's details and toggle them to `active = true`, which allows the user to log in via Supabase Auth.
- **Suspensions**: Admins can revoke access at any time by toggling `active = false`.

## 2. Zone Management
- **Route**: `/admin/zones`
- **Action**: Admins create, edit, or delete geographical zones. Users are subsequently assigned to these zones.

## 3. Reviewing Zone Reports
- **Route**: `/admin/zone-reports`
- **Action**: Admins can impersonate or directly view any submitted report for any zone. Because Admins bypass RLS (Row Level Security), they have full read access to all entries.

## 4. City Report & Aggregation
The City Report is the core administrative feature, aggregating data from all zones.
- **Route**: `/admin/city-report`
- **Action**: Admins select a period. The system fetches the data dynamically generated by the PostgreSQL Views.
- **Exporting**: Admins can export the full City Report to customized PDF or Excel formats.

## 5. The Override System
Occasionally, mathematical aggregations must be manually adjusted for the official City Report.
- **Action**: From the `/admin/city-report` view, admins can click any aggregated field and input an override value.
- **Mechanism**: This saves a record to the `city_report_overrides` table (recording `year`, `month`, `section`, `field`, `value`).
- **Effect**: The PostgreSQL View instantly registers the override and serves the overridden value instead of the calculated sum. This preserves the original data submitted by Zone Managers while correcting the final output.
